# 本地模型 vs API 选型

## 项目要求

对比开源本地模型与在线 API 模型的选型维度，从成本、隐私、延迟、定制化能力做综合分析。

## 核心要点

1. 开源本地模型特点
2. 在线 API 模型特点
3. 成本对比分析
4. 隐私对比分析
5. 延迟对比分析
6. 定制化能力对比分析

## 参考实现

1. **创建并激活虚拟环境（推荐）**

### 模型假设与局限

#### 成本模型假设

- **本地部署**：采用阶跃式硬件摊销模型，根据日均请求量选择4个硬件层级（入门GPU/中端GPU/高端GPU/多GPU集群）。月度成本 = 硬件摊销 + 24小时运行电费（电价假设$0.12/kWh）。适用于长期稳定负载场景，短期波动可能导致成本估算偏差。

- **API部署**：线性按token计费模型，支持GPT-3.5/4、Claude 3等预设定价，也可自定义单价。假设每月30天，无最低消费或阶梯定价。实际账单可能包含请求次数费、超时重试等额外成本。

#### 隐私等级规则

- **离散等级**：高(High)/中(Medium)/低(Low)三档。
- **判定规则**：数据离境(+2风险分)、第三方日志留存(+2风险分)、外部端点(+1风险分)。0分=高隐私，1-2分=中等，3+分=低隐私。
- **API默认**：数据必离境、必经过第三方日志、端点必为外部，因此固定为低隐私等级。

#### 延迟估算假设

- **所有延迟值均为占位假设**，非实际测量数据。
- **本地模型**：假设每token处理时间20ms，P95 = 均值 × 1.5。不含网络开销。
- **API模型**：假设RTT 100ms + 排队等待50ms + 每token处理30ms，P95 = 固定开销 + (token处理 × 2.0)。

#### 定制化评分权重

- 权重集中在 `constants.py` 的 `CUSTOMIZATION_WEIGHTS` 常量中：
  - 需要微调能力：40%
  - 可自托管权重：35%
  - 可固定输出Schema：25%
- **本地模型**：三项能力均为满分(1.0)。
- **API模型**：不可自托管权重(0分)；无微调需求时该项得满分，有需求时得0分；固定输出Schema能力取决于场景配置。
